# Contributing to Boundary Protocol

Thank you for your interest in contributing to the Boundary Protocol.

Boundary Protocol is a specification repository. Contributions should focus
on improving the clarity, correctness, and completeness of the protocol
definition rather than implementing features.

---

# Contribution Principles

Contributions to this repository should follow three principles:

1. **Clarity over complexity**

   The protocol is intentionally minimal. Contributions should not introduce
   unnecessary complexity.

2. **Specification stability**

   Core protocol concepts must remain stable once defined.

3. **Accountability first**

   The protocol exists to describe accountability structures across system
   boundaries. Contributions should reinforce this goal.

---

# Types of Contributions

The following types of contributions are welcome:

### Documentation improvements

Examples:

- clarifying explanations
- fixing inconsistencies
- improving examples
- correcting terminology

### Specification clarifications

Examples:

- improving model descriptions
- clarifying protocol boundaries
- refining schema documentation

### Example improvements

Examples:

- additional example traces
- better example artifacts
- clearer example event chains

---

# Changes to Core Protocol Concepts

Changes affecting the following areas require careful review:

- CORE models
- CONTRACT definitions
- SCHEMA structures
- VALIDATION rules
- CONSTITUTION principles

These areas define the protocol itself.

Proposals that change protocol semantics should include:

- clear motivation
- explanation of impact
- compatibility considerations

---

# Pull Request Guidelines

When submitting a pull request:

1. Provide a clear description of the change
2. Explain the motivation
3. Reference related documentation if relevant
4. Keep changes focused and minimal

Small, well-scoped pull requests are preferred.

---

# Protocol Scope

Boundary Protocol intentionally **does not define**:

- system orchestration
- execution engines
- control frameworks
- AI agent architectures

These systems may use the protocol, but they are outside the scope of the specification.

---

# Code of Conduct

Please be respectful and constructive in all discussions.

Boundary Protocol aims to support a broad ecosystem of systems and tools.
